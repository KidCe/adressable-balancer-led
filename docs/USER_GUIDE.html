In standby, external LEDs stay off and one mirrored LED pair gives a very dim, slow heartbeat. This keeps the chosen color identifiable while greatly reducing LED power.

Keep holding instead of releasing to use the normal 5-second calibration or 10-second reset.

Normal checker display

The live battery bar stays very dim to save power. Hold either button to make it gradually brighter for viewing in sunlight; release to return immediately to the power-saving level.

Low-voltage warning

Below 3.45 V the bar dims further. At 3.40 V it changes to a red blink that gradually grows to full brightness over two minutes.

Quick troubleshooting

Buttons do nothing: use the three-click unlock.

Battery bar seems wrong: recalibrate at full charge.

Checker mode did not start: hold the button from power-on through the complete startup display and for two more seconds.

Optional external LEDs

Need the same lighting on a larger build? The onboard pattern can continue along an external addressable LED chain without changing how the two buttons work.

The data output supports up to 64 LEDs. The 14 onboard LEDs remain the main 7+7 mirrored light; positions 15–64 repeat that complete onboard pattern.

Power note: external LEDs connect to the data, +V and GND pads and run from the same 1S voltage as the onboard LEDs. Small chains do not require a separate supply; large strips may still need additional wiring for current handling.

Power & external connections

In typical racing use the extra current draw from the LEDs is imperceptible and effectively negligible compared to motor draw. Example: a 6S 1500 mAh race pack emptied in ≈90 s implies average flight power around 1.5 kW, while 14 full-white LEDs consume ≈4 W (~0.25% of flight power).

Power & external connections

This board is intended to run from a single-cell (1S) LiPo. For racing use the motor currents and rapid battery discharge make LED draw negligible for flight performance — running the lights from 1S is expected and supported.

You can attach external addressable LEDs to the three solder pads (data, +V, GND). The board provides the data line; large external LED chains must be powered from an appropriate external supply and share ground with the board.

Control, buttons and lock

All features are accessible from the primary button; the second button on some PCBs may be disconnected due to manual hardware changes. The button lock prevents accidental changes caused by vibration — unlock with three quick clicks.

Colors and profiles

The color set follows standard FAI/Betaflight assignments (R1, R2, F2, F4, R7, R8) with two additional colors for low-band 6 and 7. The guide describes the basic principle; describing the mirrored 7+7 layout is unnecessary detail for normal operation.

Flashing & programming

Programming uses a 1.27 mm pogo-pin connector. Four pins (SWCLK, SWDIO, GND, VCC) are the minimum for SWD programming; a 7-pin connector is useful if you want reset and UART (RX/TX) for features like a future UART bootloader or Betaflight passthrough.

Common tools (J-Link OB, ST-Link) work for flashing. A low-cost J-Link OB V8 with USB-C has been used successfully for testing.

A small hint for the curious: the buttons may respond to more than ordinary taps and long holds. Rhythm matters—but some features are more fun to discover than to document.
